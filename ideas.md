# Activity Ideas & Project Directions

This sprint uses a simple four-phase flow:

1. **Learn** — run one or more Granite cookbook recipes.
2. **Brainstorm** — pick a practical agent or tooling direction.
3. **Develop** — build and test a scoped contribution.
4. **Present** — demo results and lessons learned on closing day.

---

## Suggested Activity Plan (April 12–18)

### Day 1: Learn + Brainstorm
- Walk through [granite-agent-cookbook](https://github.com/ibm-granite-community/granite-agent-cookbook).
- Reproduce at least one recipe end-to-end.
- Draft a one-page project brief: user, problem, scope, and demo plan.

### Days 2–6: Develop + Iterate
- Implement your MVP agent/workflow.
- Get mentor feedback during daily office hours.
- Produce at least one meaningful artifact per team (PR, issue, docs, prototype).

### Day 7: Present
- Share problem, approach, Granite integration, and outcomes.
- Call out what worked, what failed, and what is next.

---

## Project Directions (High Fit)

### 1) Granite Agent Workflows
- Task-planning and execution agent using LangGraph patterns.
- Multi-step coding helper (plan, implement, test, summarize).
- Documentation triage agent for open-source repos.


---

## Scoping Guardrails

- Target one demo-ready workflow, not a full product.
- Prefer depth over breadth: one solid contribution beats many partial features.
- Keep your repo presentation-ready: setup steps, architecture note, and results.
